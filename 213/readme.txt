Сервер:
	- запуск из ком строки с опцией "-p" или "--port"
	  например: d:\>python server_1.py -p 9999
	- запуск без опций выводит хелп не запуская сервер
	- запуск с флагом "-h" или "--help" выводит хелп не запуская сервер
	- тесты через клиент при запущеном сервере


Клиент:
	- запуск из командной строки с опцией(ями) [-p, --post | -g, --get | -q, --queue]
	  например:  d:\>python client_1.py -p "1st message"
	- запуск без опций выводит хелп не отправляя ничего на сервер
	- запуск с флагом "-h" или "--help" выводит хелп
	- доктесты для првоерки основных функций клиента и сервера вшиты в основную функцию клиента
	  для проверки это самой функции нужно:
		1. Запустить сервер на 8000 порте
		2. запустить клиент с модулем doctest
		   например: d:\>python -m doctest client_1.py
		   один тест, в котором 'expectation nothing', всегда будет фейлится,  это приемлемо, т.к. он нужен для забития очереди на сервере(100 сообщений исходя из требований)
